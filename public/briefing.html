01 Situation

Customers are shopping for events, but the journey breaks when products are hard to find locally.

The brief gives us a precise executive problem: customers are shopping for upcoming events, but poor reviews show they cannot find current styles or specific items in stores. The solution needs to connect style intent, catalogue retrieval, local stock, and store execution.

Review pain

"I could not find the look in store."

Customers know the event, but not always the exact category, colour, formality, or available substitute.

Operational pain

Associates bridge too many systems.

They need to infer intent, check stock, protect budget, explain substitutions, and keep the experience moving.

Executive pain

Missed demand is invisible.

A failed wedding-guest or conference mission should become a signal for buying, allocation, or transfers.

02 Solution

RetailNext Event Stylist builds an outfit and shows why it can be fulfilled.

The app extends the OpenAI clothing RAG cookbook into an event-aware, inventory-aware stylist. OpenAI handles ambiguous customer judgement; RetailNext systems enforce price, stock, fulfilment, and business constraints.

RetrieveEmbeddings find semantically relevant catalogue candidates

What OpenAI does

Interprets the human input.

Vision analysis, structured retail intent, semantic retrieval queries, substitution explanations, chat-to-action conversion, and customer-facing copy.

What RetailNext controls

Applies the retail rules.

Only catalogue SKUs are recommended. Price, budget, local stock, fulfilment, and substitution enforcement remain deterministic application logic.

03 Generate sequence

What happens when the customer clicks Generate outfit.

This is the simplest technical walkthrough: RetailNext captures and grounds the shopping mission, while OpenAI APIs handle the ambiguous interpretation, semantic retrieval inputs, and shopper-facing language.

RetailNext owned OpenAI API OpenAI + RetailNext

RetailNext frontend

RetailNext backend

API receives request

The browser posts that shopping mission to the server. The backend resolves event, style, store, budget, urgency, and starter-product context.

OpenAI API

OpenAI structures intent

GPT-4o mini reads the starter image and returns structured JSON: item attributes, event fit, outfit roles, and suggested catalogue searches.

analyzeReferenceWithOpenAI()
4
OpenAI API + RetailNext vectors

Embeddings retrieve products

OpenAI embeds the search phrases; RetailNext compares those vectors against prepared catalogue embeddings to find semantically relevant SKUs.

text-embedding-3-large + cosine search
5
RetailNext rules

Retail rules filter results

RetailNext scores and filters by store stock, budget, urgency, product role, colour harmony, event suitability, and starter-item similarity.

candidateScore() + guardrail()
6
OpenAI copy + RetailNext facts

Basket is finalised

RetailNext calculates basket value, availability, substitutions, and KPIs; OpenAI writes grounded rationale and performs the final stylist QA review.

generateBusinessCopy() + reviewOutfitWithOpenAI()
7
RetailNext UI

UI explains the result

The customer sees outfit items, availability, AI understanding, Mira chat, and the add-to-basket prompt. The UI renders only the grounded server response.

renderResult()
Important boundary

OpenAI proposes interpretation, embeddings, rationale, and language. RetailNext owns the request flow, catalogue data, SKU eligibility, price, stock, fulfilment rules, basket state, and what is ultimately shown.

04 Architecture

The model interprets intent. The app verifies retail constraints.

Click through the flow to show where OpenAI is used and where deterministic RetailNext logic takes over.

OpenAI API powered

  • GPT-4o mini vision reads uploaded or selected starter-item images
  • Structured JSON turns styling ambiguity into item attributes, roles, and search intent
  • text-embedding-3-large creates semantic vectors for catalogue search phrases
  • Responses-style tool selection turns chat requests into explain, lookup, or preview actions
  • Generation writes shopper rationale, associate notes, Braze email copy, and final guardrail review

RetailNext owned

  • Frontend form, upload, generation trigger, result rendering, and basket state
  • Catalogue, SKU metadata, product images, prices, and store inventory truth
  • Budget, local availability, urgency, role matching, substitution, and fulfilment validation
  • Preview/apply safety so Mira cannot mutate a basket until the app validates the change
  • Business KPIs, missed-demand signal, deployment, and CRM/Braze handoff

07 Rollout

Start with high-intent event journeys, then connect live retail systems.

This is the production path I would propose after the prototype.

30 days

Pilot three event missions

Wedding guest, work event, and vacation. Measure completed baskets, availability rate, chat refinements, email sends, and customer feedback.

60 days

Connect live retail systems

PIM catalogue, POS inventory, OMS fulfilment, customer preference store, Braze triggered lifecycle messaging, and store associate workflows.

90 days

Harden and scale

Add formal evals for outfit coherence, substitution quality, hallucination prevention, latency, cost, and conversion lift by store cluster.

08 Demo path

A 20-minute onsite story tied to the original brief.

Use this path when presenting to the role-play Head of Innovation and CTO.

  1. Set the problem: event shoppers are failing because style discovery and local inventory are disconnected.
  2. Generate an outfit: choose a starter item or upload an image, select occasion, style, store, budget, and urgency.
  3. Show AI understanding: OpenAI converted the vague mission into structured retail intent and starter-item attributes.
  4. Show grounding: embeddings retrieved candidates, then application logic filtered by store, budget, urgency, and outfit completeness.
  5. Use Mira: ask for available formal shoes, select one from the availability list, then preview and apply the swap.
  6. Activate: create the associate brief or email the outfit to show the journey becoming a store and CRM workflow.
  7. Close on value: the same session improves the customer journey, the associate handoff, and the merchandising signal.
